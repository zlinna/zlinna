0001 Q. 给你一个网站，如何测试?

Oct 9th 2019

~> i. 首先查找需求、网站设计(概要和详细设计)等相关文档，分析测试需求。

~> ii. 然后制定测试计划，确定测试范围和测试策略。

测试策略为针对不同项目是否需要测试、测试所需人力、测试优先级、测试介入时间点等方面的确定。

测试范围一般包括以下部分：

  • 功能测试
  • 界面测试
  • 性能测试
  • 数据库测试
  • 安全性测试
  • 兼容性测试

功能测试

  1. 链接测试
  2. 提交post功能的测试
  3. 多媒体元素是否可以正确加载和显示
  4. 多语言是否能够正确显示选择的语言
  5. …

界面测试

  1. 页面是否风格统一、美观
  2. 页面布局是否合理，重点和热点内容是否突出
  3. 空间是否正常使用
  4. 对于必须但未安装的控件，是否提供自动下载并按照的功能
  5. 文字检查
  6. …

性能测试

  1. 压力测试
  2. 负载测试
  3. 强度测试

数据库测试

具体决定是否需要展开测试。数据库一般考虑连接性，对数据库的读写操作，数据内容的验证等方面。

安全性测试

  1. 基本的登录功能的检查
  2. 是否存在溢出错误，导致系统崩溃或权限泄漏
  3. 相关开发语言的常见安全性文体学检查，例如：SQL 注入
  4. 如果需要高级的安全性问题，确定获得专业安全公司的帮助，外包测试，或者获取支持
  5. …

兼容性测试

根据需求说明的内容，确定支持的平台组合，有：

  1. 浏览器的兼容性
  2. 操作系统的兼容性
  3. 软件平台的兼容性
  4. 数据库的兼容性

~> iii. 开展测试，并记录缺陷，合理的安排调整测试进度，提前获取测试所需的资源，建立管理体系，例如：需求变更，风险，配置，测试文档，缺陷报告，人力资源等内容。

~> iv. 定期评审，对测试(测试文档、测试用例、测试代码、测试报告、缺陷分析等)进行评估和总结，调整测试内容
